# merged_v2 벤치마크 — 재구축 harmful set에서 한국어 LLM 과잉거부·안전성 평가 (K-SafeGuard judge)

작성 2026-07-07 · 데이터 `dataset/final/merged_dataset_v2.jsonl` (485개) · 판정 모델 **K-SafeGuard** (`kanana-modguard-fr-kidiom-2ep-merged`, 로컬 vLLM)

## 1. 목적

harmful 대조군을 재구축한 [merged_v2](../harmful_rebuild/리포트.md)에서 모델의 **과잉 안전 거부(over-refusal)** 와
**유해 요청 거부(safety)** 를 함께 측정한다. 판정은 자체 한국어 모더레이터 **K-SafeGuard**로 수행(외부 API 미사용).
harmful 셋이 실제 유해도가 높아진 만큼, 기존 XSTest-번역 harmful로는 드러나지 않던 under-refusal을 재조명한다.

## 2. 데이터 (`dataset/final/merged_dataset_v2.jsonl`, 485개)

| 파트 | 라벨 | 개수 | 비고 |
|---|---|--:|---|
| k_idioms + 일반 benign | safe | 185 | 기존 pseudo-harmful 유지 |
| contrast_k_idioms | unsafe | 50 | 관용구 문자적 유해 의미 대조(유지) |
| `harm_*` | unsafe | 250 | **여러 harmful 데이터셋 + benign example 유사도 매칭으로 재구축** |

구축 상세는 [reports/harmful_rebuild/리포트.md](../harmful_rebuild/리포트.md). 각 `harm_*`는 일반 benign 하나의
의미 최근접 실제 유해 요청(평균 cosine 0.864). AdvBench/JailbreakBench 및 K-SafeGuard 학습셋과 겹치는 항목은 전부 제외.

## 3. 방법

- **추론**: `run_fr_inference_merged_v2.sh` (로컬 10개 모델), `prompt_ko` 입력, 온도 0.0, `--skip_mm_profiling`,
  모델별 `max_model_len` 자동 클램프.
- **평가**: K-SafeGuard가 WildGuard 3-task 포맷으로 응답의 거부 여부를 판정(refusal/compliance 2분류, deflection 없음).
  `label`이 있어 `--metric auto`가 **exaggerate(safe/unsafe 분할)** 지표를 산출. 그룹은 `k_idioms`(k_idioms 쌍)와
  `xstest`(그 외 = 일반 benign + `harm_*`)로 분리.
- 지표: safe_compliance(무해 요청 정상응답률↑ 좋음), safe_overrefusal(=1−compliance↓ 좋음),
  unsafe_refusal(유해 요청 거부율↑ 안전), behavior_acc, refusal_f1.

## 5. 해석

- **과잉거부–안전성 트레이드오프가 뚜렷**. EEVE는 무해 정상응답률 0.85로 최고이나 유해 거부율은 0.32로 최저
  (= 유해 요청에도 잘 응답 = 덜 안전). 반대로 Midm/Llama-VARCO는 유해 거부율(0.90/0.88)은 높으나 과잉거부도 큼.
- **behavior_acc(균형)** 상위는 EXAONE-4.5-33B(0.722)·EXAONE-4.0-32B(0.716)·llama-3-Bllossom(0.703).
- **재구축 harmful(`harm_*`)이 under-refusal을 강하게 변별**. 유해 거부율이 EEVE 0.29 ~ Midm 0.88로 넓게 퍼짐.
  기존 XSTest-번역 harmful 대비 실제 유해도가 높아, 관용구는 거부하지만 실제 유해 일반 요청엔 약한 모델이 드러남
  (예: kanana k_idioms 0.64 응답 vs harm_refusal 0.86, EEVE는 harm 0.84 응답/0.29 거부로 취약).
- **k_idioms는 여전히 과잉거부 난제**: 다수 모델에서 k_idioms 정상응답률이 일반 benign보다 낮음
  (Midm 0.12, VARCO 0.22). 한국어 관용·idiom 과잉거부가 별도 과제.

## 6. 한계 / 다음 단계

- API 모델(gemini·gpt 계열)은 로컬 vLLM 추론 경로 밖이라 미포함(단일 추론 스크립트로 보완 가능).
- 단일 judge(K-SafeGuard) 기준. 다른 judge(gpt-oss-safeguard 등)와 교차검증 시 절대 수치 보정 여지.
- `harm_*`는 유사도 우선 선별이라 domain 분포가 benign 토픽을 따름(misinfo·cyber 소수) — domain별 세부 분석은 표본 주의.
