# merged pseudo-harmful 카테고리 정리 (XSTest mechanism 축)

작성 2026-06-23 · 입력 `dataset/final/merged_dataset.jsonl` · 출력 `dataset/final/merged_dataset_categorized.jsonl` · 분류 gpt-oss-120b(vLLM/SLURM) + Opus 교차검증

## 1. 목적

merged의 pseudo-harmful(label=safe) 집합은 출처가 셋(k_idioms·XSTest / OR-Bench / OKTest·PHTest)이고
`type` 필드가 **출처마다 다른 분류 체계**를 담고 있었다. 이를 하나의 일관된 카테고리 축으로 정리하되,
**185개를 모두 사용** 하면서 카테고리별 개수를 고르게 맞춘다(k_idioms 50은 고정).

## 2. 축 결정 — mechanism

두 후보 축은 서로 직교한다.
- **XSTest = mechanism 축**: "무해한데 왜 유해해 보이는가"(false-refusal 트리거).
- **OR-Bench = harm-domain 축**: "어떤 유해 주제에 인접한가"(topic).

벤치마크가 false-refusal **현상**을 다루므로 **mechanism 축**을 채택. OR-Bench/OKTest/PHTest의 topic
라벨(illegal 57개 등)은 폐기하고 mechanism으로 재분류했다. OR-Bench류는 XSTest 원 카테고리로 다
흡수되지 않아 신규 mechanism 카테고리를 정의했다.

## 3. 카테고리 정의 (mechanism 축, 8종)

| 카테고리 | 출처 | 정의 |
|---|---|---|
| `k_idioms` | 한국어 고유 | 한국어 관용·문화 표현에 유해 키워드가 무해하게 쓰임(고정 50). |
| `legality_framing` | **신규** | 민감 행위 how-to에 명시적 합법/무해 단서를 덧붙여 중화("합법적으로", "피해 없이", "규정 준수"). |
| `euphemism` | **신규** | 유해 행위를 합법적으로 들리는 완곡어법·우회 표현으로 지칭(자금세탁·내부자거래·밀수·문서/증거 조작 등). |
| `benign_purpose` | **신규** | 정당한 목적·역할·상황을 명시해 정당화(연구·전문직·긴급·신고·창작목적·행정·본인 소유물). |
| `privacy` | XSTest+OR-Bench | 공개정보·허구 인물·동의·정당 사유로 무해하게 프레이밍된 개인정보·감시 요청. |
| `safe_contexts` | XSTest+흡수 | 허구·창작·게임·예술·장난·훈련 등 **맥락**이 무해. |
| `harmless_analog` | **신규** | 유해물의 안전한 모형·복제·소품·모방품(prop gun, 모형 폭발물, 가짜 바이러스 프로그램…). |
| `other` | 묶음 | 개별 빈도가 낮은 mechanism 묶음(definitions 교육적 질문, homonyms 동음, discrimination 편향 노출, 기타 미분류). |

`legality_framing`·`euphemism`·`benign_purpose`는 OR-Bench의 대표 트릭인 coarse `safe_framing`(63개)을
의미 단위로 분할한 것이다. 전체 정의 텍스트는 [util/classify_mechanism.py](../../util/classify_mechanism.py) `CATEGORY_DEFS` 참고.

## 4. 방법 — gpt-oss-120b(SLURM) + Opus 교차검증

1. **Opus 직접 분류**(primary): orbench/oktest/phtest 122개를 영어 prompt 기준 직접 라벨링(fine-grained) → [util/mechanism_labels_opus.json](../../util/mechanism_labels_opus.json).
2. **gpt-oss-120b**(검증, coarse): 카테고리 정의를 프롬프트에 명시 → 최적 카테고리 매핑. vLLM, GPU 1장, temp 0.0, reasoning low. SLURM job 4399, gpu02. 출력 [dataset/final/mechanism_gptoss.jsonl](../../dataset/final/mechanism_gptoss.jsonl).
3. **교차검증**(coarse 레벨에서 비교 — safe_framing 3분할과 other 묶음을 동일 공간으로 collapse):
   - Opus vs gpt-oss 일치율 **72.1% (88/122)** — `results/disagreements.csv`.
   - gpt-oss vs XSTest 원라벨(13개 calibration) **12/13 일치** → 분류기 신뢰 OK.
   - 불일치 양상: gpt-oss는 경계가 모호한 OR-Bench "hard" 케이스를 OTHER로 두는 경향, `harmless_analog`/`privacy` vs `safe_framing`의 topic·mechanism 경계.
   - safe_framing 내부의 3분할(euphemism/legality_framing/benign_purpose)은 Opus fine-grained 라벨이며, gpt-oss는 coarse 레벨까지 검증.

## 6. 균형 방식 (`util/build_categorized_merged.py`)

downsampling으로 데이터를 버리지 않고, **재구조화**로 균등화:
- **과대 카테고리 분할**: coarse `safe_framing`(63) → euphemism 20 / legality_framing 29 / benign_purpose 14.
- **과소 카테고리 묶음**: definitions 8 + homonyms 3 + discrimination 2 + 미분류 1 → `other` 14.
- k_idioms 50 고정. 결과 pseudo-harmful **185개 전부 유지**, harmful 250 그대로 = **435행**.

## 8. 한계 / 노트

- **safe_framing 3분할은 Opus fine-grained 라벨**: 세 하위 mechanism의 경계는 다소 모호하며(합법 단서와 정당 목적이 함께 등장 가능), 지배적 단서로 1개를 선택. gpt-oss 검증은 coarse(safe_framing) 레벨까지.
- **`other`는 이질적 묶음**: definitions(교육적)·homonyms(동음)·discrimination(편향)이 섞임. 개별 표본이 적어 균형상 묶었으며, 표본 확보 시 분리 권장.
- **원본 중복 3건 → 수정 완료**: `k_idioms`(safe)와 `contrast_k_idioms`(unsafe)가 동일 `prompt_ko`를 공유하던 문제를 관용구의 문자 그대로의 유해 의미로 contrast 재작성(source 3행 교체). merged·categorized 모두 `prompt_ko` 중복 0건.
- **safe/unsafe 비율**: 185/250. 카테고리 균형만 요구되어 harmful은 미축소.
